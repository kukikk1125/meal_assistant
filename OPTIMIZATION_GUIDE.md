# AI 响应延迟优化指南

## 问题分析
当前 AI 响应延迟主要原因：
1. Prompt 过长（2000+ tokens）
2. 模型响应时间长
3. Vercel 免费版 10 秒限制
4. 前端体验不佳

---

## 优化方案（按优先级排序）

### 🔴 高优先级（立即可做）

#### 1. 优化 Prompt（缩短 60%）
- 移除冗余说明
- 简化示例
- 使用更简洁的语言
- 预估可减少 30-50% 响应时间

#### 2. 前端优化（立即见效）
- 添加骨架屏加载状态
- 实现乐观更新
- 更好的错误提示和重试按钮
- 不减少实际延迟，但大幅改善用户体验

#### 3. 使用更快的模型
- 当前：`doubao-seed-2-0-mini-260215`
- 尝试：`doubao-seed-2-0-flash-260215`（如果有）
- 或其他更轻量的模型

---

### 🟡 中优先级（需要代码修改）

#### 4. 流式响应（Stream）
- 不等待完整响应，边生成边显示
- 用户感知延迟大幅降低
- 需要修改 API 路由和前端

#### 5. 缓存策略
- 缓存相似请求的结果
- 减少重复 API 调用
- 适用场景：食材替换建议

#### 6. 使用 Edge Runtime
- 将 API 部署到 Edge Functions
- 降低网络延迟
- Vercel Pro 支持更好

---

### 🟢 低优先级（长期方案）

#### 7. 任务拆分
- 将大任务拆分成小任务
- 并行或分步执行
- 例如：先解析菜名，再解析食材

#### 8. 自建服务器
- 完全控制执行时间
- 无函数超时限制
- 可选：阿里云、腾讯云、AWS

---
